# Entry Summary Update Checklist (2026-04-02)

Use this before changing the project entry summary.

## Required checks

1. README current stage snapshot still matches current stage status
2. STATUS mainline line still matches the actual active mainline
3. TODO only contains live next actions, not already-landed work
4. PROGRESS only contains actually landed milestones
5. Deferred freeze lines are still consistent with the current stage

## Update rule

If any of the four core surfaces disagree, fix them first.
Only then change the entry summary.

## One-line maintenance rule

> Entry summary updates follow consistency checks, not intuition.
