# Reopen Conditions for Future Stages (2026-04-02)

## Purpose

Define what kind of evidence is strong enough to reopen currently deferred lines.

## Reopenable lines and conditions

### providerRegion
Reopen only if:
- providerScope remains stable
- providerRegion becomes a demonstrated bottleneck
- region-scope path shows enough repeated pressure to justify bounded implementation

### selection redesign
Reopen only if:
- stale cached trust creates real ranking mistakes
- fixes cannot be contained to explain visibility or bounded refresh logic

### trust-score semantics rewrite
Reopen only if:
- a future stage explicitly chooses semantics evolution as the mainline
- current bounded fixes stop being sufficient

### explainability rewrite
Reopen only if:
- operators repeatedly fail to interpret current structured visibility
- the issue cannot be fixed with bounded wording improvements
